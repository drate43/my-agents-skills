코드베이스 탐색 스킬. **항상 절대경로:라인번호 형식**으로 결과를 제공한다.

## 출력 형식

탐색 결과는 아래 형식으로 정리:

```
### 파일 위치
- `src/경로` — 설명

### 핵심 코드 위치
| 기능 | 위치 | 확인 |
|---|---|---|
| 설명 | `/절대경로:라인` | ✓ `실제 코드` |

### 연결 흐름
/.../파일A:라인 → /.../파일B:라인 → /.../파일C:라인
```

## 핵심 규칙

1. **절대경로 필수** — 프로젝트 루트 기준 절대경로:라인번호 (예: `/프로젝트경로/src/파일경로:라인번호`)
2. **라인 번호 검증** — 표에 넣기 전 Read로 실제 확인. 확인 표에 ✓/✗ 표시
3. **한국어 응답**
4. 다중 주체 흐름(3개 이상 주체)은 ASCII 시퀀스 다이어그램 포함:
   ```
   [사용자] → [프론트:페이지] → [API Route] → [외부 API]
                    ↑ 결과 콜백                      ↓ 요청
              [결과 처리 페이지] ← ─ ─ ─ ─ ─ ─ [리다이렉트]
   ```
5. 프로젝트 구조를 먼저 파악한 뒤 탐색 (디렉토리 분리 패턴, 모바일/데스크톱 분기 인지)

## 스킬 선택 기준

| 상황 | 사용 스킬 |
|------|----------|
| 코드 위치/구조/흐름 질문 | **find-in-code** |
| 두 코드 비교·분석 | **diff-review** |
| 구현 방안 제안 | **propose-fix** |
