# Grill Me — 아이디어 심문 스킬

구현에 들어가기 전 **설계의 모든 분기를 탐색**하여 공유된 이해에 도달한다.

## 동작 방식

아래 프롬프트로 사용자를 심문한다:

> 이 계획의 모든 측면에 대해 집요하게 질문합니다.
> 설계 트리의 각 분기를 하나씩 내려가며, 결정 간 의존성을 순서대로 해소합니다.
> 코드베이스를 탐색해서 답할 수 있는 질문은 직접 탐색하여 답합니다.

## 규칙

1. **질문이 끝날 때까지 구현 방안을 제시하지 않는다** — 모든 설계 분기가 해소된 후에야 다음 단계(write-prd, plan-first 등) 제안
2. **코드베이스에서 답을 찾을 수 있으면 직접 찾는다** — 사용자에게 묻지 않음
3. **결정 간 의존성을 존중한다** — A가 정해져야 B를 물을 수 있으면, A부터 해소
4. **한 번에 1~3개 질문** — 질문 폭탄 금지, 소화 가능한 단위로
5. **진행률 표시** — 탐색한 분기 / 남은 분기를 대략적으로 알려줌

## 질문 관점

| 관점 | 예시 질문 |
|------|---------|
| 사용자 시나리오 | 어떤 사용자가, 어떤 상황에서, 무엇을 기대하는가? |
| 엣지 케이스 | 네트워크 실패, 빈 데이터, 동시 요청 시 어떻게 되는가? |
| 기존 코드 영향 | 기존 기능 중 깨질 수 있는 것은? |
| 플랫폼 분기 | iOS/AOS/웹에서 동일하게 동작하는가? |
| 데이터 흐름 | 데이터가 어디서 오고, 어디에 저장되고, 누가 소비하는가? |
| 범위 | MVP에 반드시 포함해야 하는 것 vs 나중에 해도 되는 것? |

## 완료 조건

- 모든 설계 분기가 해소됨
- 사용자와 공유된 이해에 도달함
- "다음 단계로 `write-prd` 또는 `plan-first`를 사용할 수 있습니다" 안내
