# 개발 사이클 오케스트레이터

PRD 또는 이슈 분해 결과를 받아 **계획 → 구현 → 리뷰 → 문서**까지 진행합니다.

---

## 워크플로우

```
[PRD / 이슈 분해 / 사용자 요청]
    ↓
[STEP 1] 구현 계획 수립     ← 변경 파일 목록 + 코드 스니펫 미리보기
    ↓
[STEP 2] 아키텍처 리뷰      ← senior-frontend-architect 에이전트 호출
    ↓
[STEP 3] 사용자 승인 요청   ← 계획 + 리뷰 결과 제시 후 대기
    ↓  (승인 후)
[STEP 4] TDD 구현           ← tdd 에이전트 호출
    ↓
[STEP 5] 코드 리뷰          ← senior-frontend-reviewer 에이전트 호출
    ↓
[STEP 6] 수정 반영          ← 🔴 항목 즉시 수정, 🟡 항목 사용자 확인
    ↓
[STEP 7] 문서 업데이트      ← 구현 근거/커밋 내역 기준으로 docs/ 반영
    ↓
[완료]
```

---

## 각 단계 상세

### STEP 1: 구현 계획 수립

PRD 또는 이슈 분해 결과가 있으면 그 기반으로, 없으면 직접 탐색하여 계획을 작성합니다.

```
## 작성 계획

### 1. [파일명] (신규 | 수정)
**변경 위치**: `파일경로:라인번호`
**변경 이유**: 한 줄 설명

코드 미리보기:
```ts
// 작성할 코드 스니펫
```
```

- 변경되는 **모든 파일**을 빠짐없이 나열
- 각 파일마다 변경 위치와 변경 이유 명시
- 실제 작성할 코드를 스니펫으로 미리 표시

### STEP 2: 아키텍처 리뷰

계획 작성 완료 후 **즉시** `senior-frontend-architect` 에이전트를 호출합니다.

**리뷰 결과 처리:**
- ✅ 계획대로 진행 → STEP 3으로 이동
- ⚠️ 수정 후 진행 → 계획을 수정한 뒤 STEP 3으로 이동
- 🔴 재설계 필요 → 계획을 재작성한 뒤 다시 STEP 2 수행

### STEP 3: 사용자 승인 요청

계획 + 아키텍처 리뷰 결과를 제시하고 "진행할까요?"로 승인 요청.
**사용자가 명시적으로 승인하기 전까지 절대 코드를 작성하지 않습니다.**

### STEP 4: TDD 구현

승인 후 `tdd` 에이전트를 호출하여 Red-Green-Refactor 사이클로 구현합니다.
tdd 에이전트가 없거나 테스트가 불필요한 경우, 계획대로 직접 코드를 작성합니다.

### STEP 5: 코드 리뷰

모든 파일 구현 완료 후 **즉시** `senior-frontend-reviewer` 에이전트를 호출합니다.

### STEP 6: 수정 반영

코드 리뷰 결과를 사용자에게 제시하고 처리합니다.

- **🔴 반드시 수정** 항목 → 사용자 확인 없이 즉시 수정 적용
- **🟡 개선 제안** 항목 → 사용자에게 의견을 물어 반영 여부 결정
- **💡 선택 사항** 항목 → 참고용으로만 제시, 적용하지 않음

### STEP 7: 문서 업데이트

구현 완료 후 관련 문서(`docs/`)를 **실제 반영된 내용 기준**으로 갱신.
- 근거: 커밋 내역 → diff → 실제 파일 순으로 수집
- 계획이 아니라 확정된 내용만 문서에 작성

---

## 단계 생략 옵션

사용자가 명시적으로 요청할 때만 단계 생략 가능:

| 사용자 요청 | 생략되는 단계 |
|-------------|---------------|
| "아키텍처 리뷰 없이 해줘" | STEP 2 생략 |
| "코드 리뷰 없이 해줘" | STEP 5~6 생략 |
| "바로 구현해줘" / "계획 없이 해줘" | STEP 1~2 생략 |
| "TDD 없이 해줘" | STEP 4에서 tdd 에이전트 미호출, 직접 구현 |
| "문서 업데이트는 하지마" | STEP 7 생략 |

---

## 커뮤니케이션 규칙

- 항상 한국어로 응답
- 각 단계 시작 시 현재 단계 명시: `[STEP N] 단계명`
- 코드 위치는 항상 `파일명:라인번호` 형식
- 에이전트 호출 전 알림 (예: "senior-frontend-architect 에이전트로 아키텍처를 검토합니다.")
- 완료 보고에는 코드 변경, 문서 업데이트 여부를 함께 정리
