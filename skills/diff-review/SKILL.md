# Code Compare 스킬

코드 변경을 다룰 때 **일관된 before/after 비교 형식**으로 출력한다.

## 출력 구조

파일 단위로 아래 형식을 반복한다. 변경 파일이 적으면 간결하게, 많으면 핵심 파일만 상세히 다룬다.

### 파일별 섹션
```
## `절대경로` 코드 비교
### 문제 구간: 라인 X~Y
```

**라인별 표** — 핵심 변경만 발췌:

| 라인 | **A (이전)** | **B (변경)** |
|---|---|---|
| **N** | `코드` | `코드` |
| **N+1** | _(삭제됨)_ | `새 코드` |

**코드블록 병렬 비교**:

**A** (`절대경로:시작~끝`) / **B** (`절대경로:시작~끝`)로 코드블록 2개 배치.

### 에러 전파 경로 (변경이 동작 흐름에 영향을 줄 때만)

| 단계 | A (이전) | B (변경) |
|---|---|---|
| 설명 | 동작 | **변경된 동작** |

## 규칙

1. **절대경로 필수** — `프로젝트루트/src/파일경로:라인번호` 형식. cmd+click 이동 가능해야 함
2. 브랜치 비교면 헤더에 브랜치명, 커밋/PR이면 해시/번호 명시
3. 에러 쪽은 **굵게 + CRASH/ERROR** 표기. 예: `**CRASH** — TypeError: Cannot read property 'id' of undefined`

## 일괄 변경 형식

동일 패턴을 여러 곳에 적용할 때:
1. Before/After 대표 예시 1개
2. 수정 대상 라인 목록 (표)
3. 수정 제외 목록 + 이유

## 스킬 선택 기준

| 상황 | 사용 스킬 |
|------|----------|
| 이미 존재하는 두 코드 비교 | **diff-review** |
| 아직 없는 코드를 새로 제안 | **propose-fix** |
| 구현 제안 안에 Before/After 포함 | **둘 다** |
